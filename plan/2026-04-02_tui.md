# Blobcache Browse TUI

- When started, the terminal UI attempts to render whatever namespace BLOBCACHE_NS_ROOT points to.
- The terminal UI is built using the bubbletea framework
- All mutating operations are provided by a menu which comes up when the leader key is pressed. The leader key is space.
- The leader key menu contains the key to press and a description of what action will be performed.
- hjkl and the arrow keys are used for movement.
- the tui can be started with `blobcache browse`

## Main View

The default mode is rendering the contents of a volume.
A top bar displays the current path, which is a stack of names that have been traversed.
The root ns volume FQOID is also displayed in the top left, with the path after it.
Different volumes have different schemas, and therefore: different rendering logic.
Below the top bar is between 1 and 3 horizontal panes, depending on the size of the path stack.
If there are 0 names on the stack, then there are 2 panes. One rendering the root on the left, and another rendering a preview of whatever is under the cursor on the right.
If there is 1 or more name on the stack, then the there are 3 panes, with the root rendered in the left pane, the named volume in the center, and a preview of whatever is selected in the right pane.

## Namespace View

Namespaces entries appear sorted 1 per line, and all their information is displayed in a table.
A cursor is initially positioned on the top row.
Movement keys move the cursor up and down.
Left and right move into or out.
Navigating around with left and right causes the path to be manipulated, which may lead to another namespace being rendered, or a different object entirely.
Pressing the / key brings up a search modal across the bottom of the page.

## Search Modal

- The search modal can be called to filter any table or list.
- It is summoned with /
- It allows the user to type or press escape to exit the search modal.
- The search modal is a stacked mode, so exiting it reverts to the previous mode.
- It performs substring search and only displays matches.
- Pressing enter pops the search mode from the stack and executes a arbitrary callback with the selected value passed to it. (SKIP IMPLEMENTATION)
- Pressing escape exits the search mode and returns to the previous view.

## Menu Mode

Pressing the leader key (space) in the default mode brings up a menu which contains actions.
Menu mode is a stacked mode and escape goes back to whatever view was active previously.

Actions:
- r refreshes the page.
- y copies the currently selected item to the clipboard.  What is yanked to the clipboard is schema dependent.
