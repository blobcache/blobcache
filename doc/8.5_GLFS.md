# GLFS (Git-Like Filesystem)

Utilities for working with the Git-like filesystem representation stored in Blobcache volumes.
These commands operate on volumes by name via the basic namespace.

Note: The `<volume>` argument refers to a volume name in `basicns`, not a raw handle.

## `blobcache glfs init <volume>`
Initializes an empty GLFS tree in the given volume.
Fails if the volume already contains data.

## `blobcache glfs look <volume> <src>`
Looks up a path in the volume and prints information.
If the path points to a tree, prints entries (name, mode, ref).

### Positional Arguments
- `volume` The basicns volume name to open.
- `src` The GLFS path to inspect (use `/` as the root).

## `blobcache glfs import <volume> <dst> <src>`
Imports data from a local filesystem directory `src` and merges it under the GLFS path `dst` in the target volume.

### Positional Arguments
- `volume` The basicns volume name to open.
- `dst` The destination GLFS path inside the volume (e.g., `/docs`).
- `src` The local filesystem source directory to import.

## `blobcache glfs read <volume> <src>`
Reads a file at GLFS path `src` and writes its contents to stdout.
Errors if `src` is not a blob.

### Positional Arguments
- `volume` The basicns volume name to open.
- `src` The GLFS file path (e.g., `/README.md`).

## `blobcache glfs sync <src> <dst>`
Efficiently sets the contents of the destination volume to match the source volume.

### Positional Arguments
- `src` The source basicns volume name.
- `dst` The destination basicns volume name.
