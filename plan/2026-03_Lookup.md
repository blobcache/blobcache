# 2026-03-10 Namespace Lookup

Add Lookup functionality to the schema/bcns package.

Right now namespaces (defined in bcns) can only contain alphanumerics, dashes, and underscores.
We will change this rule (defined in a regexp) as follows:
1. Names must start with a letter.
2. Names can contain slashes, numbers, underscores, and dashes.
3. Names cannot end in a slash, underscore, or dash.

Lookup on a single namespace volume is defined as follows:
If there is an entry in the namespace with the target name, then return that entry.
If there is not, then split the name along slashes and throw away the last element.
Trim slashes so they are not the beginning or end.
If the new name is the empty string, then return an error (no entry found).
If the new name is also not in the namespace then split along slashes and repeat.
If any one of the prefixes has a corresponding entry, that is a match.
The entry is returned from Lookup along with the suffix, which could not be matched.
This means the longest prefix and shortest suffix will be preffered to shorter prefixes and longer suffixes.

The global lookup works as follows.
Starting at whatever volume, iteratively lookup the remainder until an error or an entry and empty remainder are returned.
An entry and empty remainder are a success.

Unlike Get, which handles missing values without an error, Lookup should return an error if the lookup cannot be resolved to an entry without a remainder.
